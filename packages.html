How DPM Works

The Djazair Package Manager (DPM) uses a modern, decentralized architecture. Packages are fetched directly from the official extensions repository or any public GitHub repository. When you install a package, DPM downloads it securely into your global Djazair libraries directory, making it instantly available to all your scripts.

Install Packages
Install globally from name, GitHub, or ZIP files.
dpm install <pkg>
Build Native Code
Compile a package's native C/C++ extensions dynamically.
dpm build <pkg>
Update Dependencies
Update installed packages from their original sources.
dpm update <pkg>
Package Scaffold
Initialize a new package and generate dpm.json.
dpm init

Contribute an Extension

The Djazair ecosystem thrives on community contributions. If you have built an awesome module, wrapper, or C/C++ extension, you can easily share it with everyone by submitting it to the official repository.

Create a dpm.json: Define your package name, version, and details using dpm init.
Fork & Commit: Fork the djazair-extensions repository and add your package folder.
Open a PR: Submit a Pull Request. Once approved, it will be instantly available to all developers!
